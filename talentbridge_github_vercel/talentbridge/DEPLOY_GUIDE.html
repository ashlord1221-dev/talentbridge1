Deploy TalentBridge to Render

Complete step-by-step guide — your site will be live in ~10 minutes

STEP 1

Create a GitHub Account FREE

GitHub stores your code and connects to Render for automatic deployment.

🌐
Go to github.com → Click "Sign up" Enter email, password, username → Verify your email
→ Open github.com/signup
✓ Already have a GitHub account? Skip to Step 2.
STEP 2

Create a New Repository

A repository is where your project code lives on GitHub.

1️⃣
Click the "+" icon (top right) → "New repository"Or go to: github.com/new
2️⃣
Repository name: talentbridgeKeep it lowercase, no spaces
3️⃣
Set to "Public" → Click "Create repository"Public repos are free on GitHub
STEP 3

Upload Your Project Files

Upload all files from the downloaded ZIP to your GitHub repository.

📦
Extract (unzip) the downloaded fileYou'll get a folder called "talentbridge" with app.py, templates/, etc.
📤
On GitHub: click "uploading an existing file" You'll see this link on the empty repository page
🗂️
Drag ALL files into the upload area Select everything inside the talentbridge folder:
app.py · requirements.txt · Procfile · render.yaml · README.md · .gitignore
templates/ folder · static/ folder · .github/ folder
✅
Click "Commit changes" (green button at bottom)Leave the commit message as default, click the green button
⚠️ Make sure you upload files from INSIDE the talentbridge folder — not the folder itself. GitHub should show app.py at the root level.
STEP 4

Create a Render Account FREE

Render hosts your Flask app online for free.

🌐
Go to render.com → Click "Get Started for Free" Sign up with your GitHub account (easiest — one click!)
→ Open render.com
✓ Sign up with GitHub so Render can access your repositories automatically.
STEP 5

Deploy on Render AUTO-DEPLOY

Connect your GitHub repo to Render and deploy with one click.

1️⃣
Click "New +" → "Web Service"From the Render dashboard
2️⃣
Select "Build and deploy from a Git repository"Then click "Next"
3️⃣
Find your "talentbridge" repo → Click "Connect"If you don't see it, click "Configure account" to give Render access
4️⃣
Fill in these exact settings:

Render Settings

FieldValue
Nametalentbridge
RegionSingapore (closest to India)

PlanFree
5️⃣
Click "Create Web Service" (blue button)Render will now build and deploy your app — takes 3–5 minutes
STEP 6

Enable Auto-Deploy (GitHub Actions) OPTIONAL

Every time you push code to GitHub, it automatically deploys to Render.

1️⃣
In Render dashboard → your service → Settings → Deploy HookCopy the "Deploy Hook URL" — it looks like: https://api.render.com/deploy/srv-xxx...
2️⃣
In GitHub → your repo → Settings → Secrets → ActionsClick "New repository secret"
3️⃣
Add this secret: Name: RENDER_DEPLOY_HOOK_URL
Value: (paste the URL you copied from Render)
✓ Now every git push → GitHub runs tests → if tests pass → auto-deploys to Render!
STEP 7

Watch It Deploy

Monitor the build logs in Render. It takes about 3–5 minutes.

📋
In Render — click your service → "Logs" tabYou'll see lines like: "Installing packages", "Starting gunicorn"
✅
When you see "Your service is live" — you're done!Render gives you a URL like: https://talentbridge.onrender.com
Booting worker with pid: 123 [INFO] Starting gunicorn 21.2.0 [INFO] Listening at: http://0.0.0.0:10000 ✓ Your service is live 🎉
